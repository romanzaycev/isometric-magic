## 8. Добавить сбор технических метрик кадра в `FrameStats`

**В чем проблема**
Сейчас видны базовые метрики вроде имени сцены, backend, viewport и завершения кадра, но нет данных, которые позволят быстро локализовать bottleneck: количество update-компонентов, обходов сцены, draw calls, texture binds, загрузок текстур и GC alloc/frame.

**Как решаем**
Расширяем `FrameStats`, чтобы он собирал показатели CPU и render pipeline, а не только общие данные кадра.

**Предлагаемый путь решения**

* Вариант A: добавить только CPU-метрики по этапам кадра.
* Вариант B: добавить только render-метрики.
* Вариант C: добавить оба набора, но включать подробную статистику только в debug/development режиме.

**Что получим после решения**
Появится объективная база для оптимизаций: будет видно, что реально тормозит, а что нет.
